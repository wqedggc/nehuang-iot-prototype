# 涅凰智农 — 绑定流程重构 PRD

> 版本：v1.0 | 作者：许清楚 | 日期：2026-07-01

---

## 一、产品目标

**问题现状：** 当前管理端「新增绑定」弹窗让管理员手填 gateway_id / user_id / freq / devid / func，与后端 POST /api/v1/admin/bindings 只接受 `user_id + point_id` 的数据模型不匹配。雪花 ID 用 number input 展示、手工填写，导致绑定后数据不一致或列表为空。

**目标：** 以 `device_points.point_id` 为核心重构绑定流程，管理员全程「选择」而非「手填」，所有 ID 按字符串处理，消除数据不一致问题。

**范围：**
- 管理端 3 个页面重构：用户管理、设备管理、绑定管理
- 新增 4 个后端接口 + 1 个保留接口
- C 端无变更（仅验收数据一致性）

## 二、用户故事

| # | 角色 | 场景 | 期望 |
|---|------|------|------|
| US-1 | 管理员 | 在用户列表看到某农户，想为他绑定新设备 | 点击农户 → 进入详情页 → 点「绑定设备」→ 搜索并选择一个设备点 → 提交，绑定立即生效 |
| US-2 | 管理员 | 不确定某个功能点是否已被绑定 | 搜索设备点时看到绑定状态（已绑定/未绑定），已绑定的显示归属农户 |
| US-3 | 管理员 | 误操作绑错了设备 | 在绑定管理页找到该记录，点击解绑，确认后解除 |
| US-4 | 管理员 | 需要编辑设备的业务标签 | 在设备管理页找到功能点，编辑显示名称、棚号、区域、备注 |
| US-5 | 管理员 | 想知道某个农户绑了多少设备 | 在用户详情页直接看到已绑定设备列表 |
| US-6 | 农户 | 登录 C 端查看自己的设备 | 绑定后立即在「我的设备」中看到新设备 |

## 三、需求池

### P0（本迭代必须完成）

| ID | 需求 | 说明 |
|----|------|------|
| P0-1 | 用户列表 → 农户详情页 | 点击农户行跳转详情，展示基本信息 + 已绑定设备列表 + 「绑定设备」按钮 |
| P0-2 | 设备点搜索弹窗 | 关键词搜索 device_points，展示 point_id / 功能类型 / 频点 / 设备ID / 棚号 / 云边设备名 / 绑定状态，已绑定显示归属农户名 |

| P0-7 | 农户详情接口 | GET /api/v1/admin/farmers/{user_id} |
| P0-8 | 农户绑定列表接口 | GET /api/v1/admin/farmers/{user_id}/bindings |
| P0-9 | 设备点搜索接口 | GET /api/v1/admin/device-points?keyword=&bind_status=unbound |
| P0-10 | 绑定列表接口（改造） | GET /api/v1/admin/bindings 支持按 user_id / point_id / status 筛选 |

| ID | 需求 | 说明 |
|----|------|------|

- keyword — 搜索关键词（匹配 display_name / gateway_name / shed_no / device_external_id / function_type 名称 / remark）
- bind_status — unbound（仅未绑定）/ bound（仅已绑定）/ 不传（全部）

### 5.4 创建绑定

POST /api/v1/admin/bindings
Body: `{"user_id":"...","point_id":"..."}`

Response 201:
```json
{
  "code": 0,
  "data": {
    "bind_id": "9999999999999999999",
    "user_id": "1234567890123456789",
    "point_id": "9876543210987654321",
    "created_at": "2026-07-01T10:00:00Z"
  }
}
```

Error 409（已绑定给其他农户）:
```json
{"code": 409, "message": "该设备点已绑定给农户「李四果园」"}
```

Error 409（已绑定给同一农户）:
```json
{"code": 409, "message": "该设备点已绑定给当前农户"}
```

### 5.5 解绑

DELETE /api/v1/admin/bindings/{bind_id}

Response 200:
```json
{"code": 0, "data": {"deleted": true, "bind_id": "9999999999999999999"}}
```

Error 404:
```json
{"code": 404, "message": "绑定关系不存在"}
```

### 5.6 绑定列表（改造）

参数（均可选）：
- user_id — 按农户筛选

- keyword — 按农户名或设备名模糊搜索
- status — active（有效）/ all（全部含已解绑），默认 active
